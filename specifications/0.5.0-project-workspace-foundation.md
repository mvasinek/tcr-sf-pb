# Feature 0.5.0 – Project Workspace Foundation

## Cíl

Vytvořit základní architekturu projektu pro dlouhodobý vývoj aplikace.

Od verze **0.5.0** již nebude aplikace pouze knihovnou analytických funkcí, ale stane se **projektově orientovaným bioinformatickým prostředím**.

Každá analýza bude reprezentována samostatným projektem.

Projekt bude obsahovat pouze metadata, konfiguraci a výsledky analýz.

Vstupní datasety budou sdílené mezi více projekty.

# Architektonické principy

Projekt bude postaven na následujících principech.

## 1. Workspace

Aplikace pracuje nad jedním workspace.

Workspace obsahuje:

* datasety
* projekty
* cache
* nastavení

---

## 2. Dataset

Dataset představuje jednu veřejnou nebo lokální studii.

Například:

```text
GSE160097

GSE123456

RA_dataset

JIA_dataset
```

Dataset existuje pouze jednou.

Může být použit více projekty.

---

## 3. Project

Projekt reprezentuje jednu analýzu.

Projekt obsahuje:

* konfiguraci
* parametry
* výsledky
* logy
* obrázky

Nikdy neobsahuje kopii datasetu.

---

## 4. Adapter

Každý dataset je zpřístupněn pomocí adapteru.

Adapter převádí data do jednotného interního modelu.

Analytické moduly nikdy nepracují přímo s formátem datasetu.

---

## 5. Analysis modules

Analýzy pracují pouze nad jednotným interním modelem.

GUI nikdy nesmí obsahovat analytickou logiku.

GUI pouze volá knihovní funkce.

# Struktura datasetu

Každý dataset:

```text
datasets/

    GSE160097/

        dataset.yaml

        raw/

        intermediate/
```

`raw/`

obsahuje původní data.

`intermediate/`

obsahuje standardizované tabulky vytvořené adapterem.

---

# Struktura projektu

Každý projekt:

```text

    JIA_Pilot/

        project.yaml

        outputs/

        figures/

        logs/

        cache/
```

---

# project.yaml

Implementuj datový model:

```yaml
project:

  name:

  description:

  created:

  modified:

tool:

  version:

datasets:

  - id:

adapter:

analysis:

status:
```

Příklad:

```yaml
project:

  name: JIA Pilot

  description: Pilot analysis of GSE160097

  created: 2026-06-16

tool:

  version: 0.5.0

datasets:

  - GSE160097

adapter:

  tenx

analysis:

  detection

status:

  extract_annotations: done

  unified_table: done

  paired_detection: done

  expansion: done

  roc_auc: pending
```

---

# dataset.yaml

Každý dataset bude mít metadata.

```yaml
dataset:

  id:

  title:

  source:

  adapter:

  created:

files:
```

Příklad:

```yaml
dataset:

  id: GSE160097

  source: GEO

  adapter: tenx

files:

  raw: raw/

  intermediate: intermediate/
```

---

# Kritéria dokončení

Feature je dokončena pokud:

* existuje Workspace API,
* existuje Project API,
* existuje Dataset API,
* existuje Manifest API,
* existuje adapter architektura,
* existuje standardizovaný interní model,
* existuje základ Streamlit GUI,
* projekt podporuje více projektů nad jedním datasetem,
* projekt je připraven na kombinaci více datasetů,
* GitHub Actions procházejí,
* všechny testy procházejí,
* verze projektu je `0.5.0`,
* po dokončení bude vytvořen git tag:
