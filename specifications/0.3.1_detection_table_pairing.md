# Hotfix 0.3.1 – Fix SF/blood pairing key

## Cíl

Opravit feature `0.3.0`.

Aktuální implementace páruje klony podle:

```text
sample_group + patient + cell_type + clonotype_key

To je špatně, protože v datasetu GSE160097 může mít stejný pacient v SF a blood odlišné sample_group.

Nově páruj podle biologického pacienta:

patient + cell_type + clonotype_key

Práce se sample_group

Protože sample_group není párovací klíč, uchovej jej pouze informačně.

Vytvoř dva sloupce:

sf_sample_group
blood_sample_group

Pravidla:

pro SF hodnoty ukládej do sf_sample_group,
pro blood hodnoty ukládej do blood_sample_group,
pokud existuje více hodnot pro stejný patient + cell_type + clonotype_key + compartment, spoj unikátní hodnoty přes středník,
hodnoty seřaď deterministicky abecedně.

Příklad:

PM1;PM3

Pokud klon v daném kompartmentu chybí, ponech prázdnou hodnotu.

Frakce

Frakce počítej nově podle:

patient + cell_type + compartment

nikoliv podle sample_group.

Tedy:

sf_fraction = sf_cells / total SF cells for patient + cell_type
blood_fraction = blood_cells / total blood cells for patient + cell_type
CLI

CLI zůstává stejné:

python -m tcr_bcr_tools.build_detection_table \
  --input ./outputs/clone_counts.csv \
  --output ./outputs/paired_detection_table.csv

Stávající parametry musí fungovat beze změny:

--cell-type
--sample-group
--min-cells

Pozor: --sample-group zůstává pouze vstupní filtr před párováním, nikoliv párovací klíč.

CHANGELOG

Doplň:

## [0.3.1] - Fix paired detection grouping key

### Fixed
- Changed SF/blood clone pairing key from `sample_group + patient + cell_type + clonotype_key` to `patient + cell_type + clonotype_key`.
- Prevented different `sample_group` values from blocking shared clone detection.
- Added separate `sf_sample_group` and `blood_sample_group` metadata columns.
- Updated fraction calculation to use patient-level compartment totals.
Kritéria dokončení

Hotfix je hotový, pokud:

klony ze SF a blood stejného pacienta se spárují i při různém sample_group,
shared_clone už může být True,
výstup obsahuje sf_sample_group a blood_sample_group,
frakce jsou počítány na úrovni patient + cell_type + compartment,
všechny testy projdou přes pytest,
projekt má verzi 0.3.1.
